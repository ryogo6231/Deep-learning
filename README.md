## 1. ディープラーニングとは

ニューラルネットワークの**隠れ層を増やす**ことで、層は深くなります。  
こうした深い層を持つネットワークのことを**ディープニューラルネットワーク**（Deep Neural Network, DNN）と呼びます。  
DNNを学習する手法のことを総じて**ディープラーニング**あるいは**深層学習**と呼びます。

### なぜ深くするのか

浅いネットワークでも理論上は任意の関数を近似できますが、層を深くすることで

- より**複雑な特徴**を段階的に学習できる
- 少ないパラメータで**高い表現力**を得られる
- 画像・音声・言語など多様なデータに対応できる

という利点があります。

### 学習の仕組み

DNNは**誤差逆伝播法**（Backpropagation）と**勾配降下法**を組み合わせて学習します。

## 2. ディープラーニングモデルの種類

### 2.1 畳み込みニューラルネットワーク（CNN）

**Convolutional Neural Network** — 主に画像認識に使われるモデル。

**特徴**  
フィルタ（カーネル）を入力上でスライドさせる**畳み込み演算**によって、局所的な特徴（エッジ・形・テクスチャ）を段階的に抽出する。

$$
Y[i,j] = \sum_{m}\sum_{n} X[i+m,\, j+n] \cdot K[m,n]
$$

この式は「フィルタを入力画像の各位置に重ねて積和をとる」操作を表しています。

| 記号 | 意味 | 役割 |
|------|------|------|
| $X[i+m,\, j+n]$ | 入力画像の画素値 | 位置 $(i,j)$ を起点にフィルタサイズ分の領域を参照する |
| $K[m,n]$ | フィルタ（カーネル）の重み | エッジ検出・ぼかしなど特定の特徴を検出するパターン |
| $Y[i,j]$ | 出力（特徴マップ）の値 | フィルタと入力が「どれだけ似ているか」を表すスコア |
| $\sum_{m}\sum_{n}$ | フィルタ全領域の総和 | フィルタ内の全画素で積和をとる |

> **直感的な理解**  
> 虫眼鏡（フィルタ）を画像の上でずらしながら、「この場所にエッジはあるか？」を数値で評価するイメージです。  
> フィルタの種類によって検出できる特徴（縦線・横線・曲線など）が変わります。

**主なレイヤー**

| レイヤー | 役割 |
|----------|------|
| 畳み込み層（Conv） | フィルタで局所特徴を抽出 |
| プーリング層（Pooling） | 特徴マップを縮小・圧縮 |
| 全結合層（FC） | 最終的なクラス分類 |

**代表的なモデル**：VGG / ResNet / EfficientNet

```python
import torch.nn as nn

class CNN(nn.Module):
    def __init__(self):
        super().__init__()
        self.conv1 = nn.Conv2d(1, 32, kernel_size=3, padding=1)
        self.conv2 = nn.Conv2d(32, 64, kernel_size=3, padding=1)
        self.pool  = nn.MaxPool2d(2, 2)
        self.relu  = nn.ReLU()
        self.fc    = nn.Linear(64 * 7 * 7, 10)

    def forward(self, x):
        x = self.pool(self.relu(self.conv1(x)))
        x = self.pool(self.relu(self.conv2(x)))
        x = x.view(x.size(0), -1)
        return self.fc(x)
```

### 2.2 リカレントニューラルネットワーク（RNN）

**Recurrent Neural Network** — テキスト・音声・時系列データに使われるモデル。

**特徴**  
前の時刻の**隠れ状態** $h_{t-1}$ を次のステップへ引き継ぐことで、過去の情報を保持しながら系列を処理する。

$$
h_t = \tanh(W_h h_{t-1} + W_x x_t + b)
$$

この式は「過去の記憶と現在の入力を合わせて、新しい隠れ状態を作る」操作を表しています。

| 記号 | 意味 | 役割 |
|------|------|------|
| $x_t$ | 時刻 $t$ の入力 | その時刻に与えられるデータ（例：文中の単語） |
| $h_{t-1}$ | 前の時刻の隠れ状態 | 過去の文脈・履歴を圧縮して保持したベクトル |
| $h_t$ | 現在の隠れ状態 | 過去と現在を統合した新しい記憶 |
| $W_x$ | 入力の重み行列 | 入力をどう変換するかを学習するパラメータ |
| $W_h$ | 隠れ状態の重み行列 | 過去の記憶をどう引き継ぐかを学習するパラメータ |
| $b$ | バイアス | 出力を調整する定数項 |
| $\tanh$ | 活性化関数 | 出力を $(-1, 1)$ に収め、非線形変換を加える |

> **直感的な理解**  
> 文章を一単語ずつ読むとき、前の単語の文脈（$h_{t-1}$）を頭に入れながら次の単語（$x_t$）を理解するイメージです。  
> $\tanh$ によって値が発散せず安定して伝わります。

**問題点と発展モデル**

長い系列では**勾配消失**が発生し、長距離の依存関係を学習できなくなります。  
これを解決するために以下のモデルが提案されました。

| モデル | ゲート | 特徴 |
|--------|--------|------|
| RNN | なし | 構造がシンプル |
| LSTM | 入力・忘却・出力（3つ） | セル状態で長距離依存を記憶 |
| GRU | リセット・更新（2つ） | LSTMより軽量 |

**代表的な用途**：機械翻訳 / 文章生成 / 音声認識

```python
class LSTMModel(nn.Module):
    def __init__(self, vocab_size, embed_dim, hidden_dim):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim)
        self.lstm = nn.LSTM(embed_dim, hidden_dim, batch_first=True)
        self.fc   = nn.Linear(hidden_dim, vocab_size)

    def forward(self, x, hidden=None):
        emb = self.embedding(x)
        out, hidden = self.lstm(emb, hidden)
        return self.fc(out), hidden
```

### 2.3 トランスフォーマーモデル

**Transformer** — 現代のAIの中核をなすモデル。2017年にGoogleが発表。

**特徴**  
RNNのように順番に処理するのではなく、**Attention機構**によって系列全体を一度に処理します。  
これにより長距離の依存関係を効率よく学習でき、並列計算も可能になりました。

$$
\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right)V
$$

この式は「各トークンがほかのどのトークンにどれだけ注目すべきかを計算し、その重みで情報を集約する」操作を表しています。

| 記号 | 意味 | 役割 |
|------|------|------|
| $Q$（Query） | 「何を知りたいか」を表すベクトル | 注目先を探すための問い合わせ |
| $K$（Key） | 「自分はどんな情報か」を表すベクトル | Queryとの関連度を計算するための鍵 |
| $V$（Value） | 「実際に渡す情報」を表すベクトル | 注目度に応じて加重平均される中身 |
| $QK^\top$ | QueryとKeyの内積 | 2つのトークンがどれだけ関連しているかのスコア |
| $\sqrt{d_k}$ | スケーリング係数 | 次元数 $d_k$ が大きいと内積が大きくなりすぎるため、正規化する |
| $\text{softmax}(\cdot)$ | 正規化関数 | スコアを合計1の注目度（重み）に変換する |
| $(\cdots)V$ | Valueの加重平均 | 注目度の高いトークンの情報を多く集める |

> **直感的な理解**  
> 図書館で「機械学習の本を探したい」（Query）→ 本の背表紙タイトル（Key）を見て関連度を判断 → 中身（Value）を読むイメージです。  
> 文中のすべての単語同士でこの操作を同時に行うことで、離れた単語間の関係も一度に捉えられます。

**アーキテクチャ**

```
入力
 └─ Embedding + Positional Encoding
      └─ Encoder（Multi-Head Attention → Feed Forward）× N層
           └─ Decoder（Masked Attention → Cross Attention → FF）× N層
                └─ 出力
```

**代表的なモデル**

| モデル | 開発元 | 用途 |
|--------|--------|------|
| BERT | Google | 文章の理解・分類 |
| GPT | OpenAI | 文章生成 |
| ViT | Google | 画像認識 |
| Whisper | OpenAI | 音声認識 |
